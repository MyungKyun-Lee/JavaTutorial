정리질문
1. 클라이언트가 request를 할 때 공부해야 하는 것 3가지
-> Method / URL / Parameter

2. url의 구성하는 각 부분을 설명하시오.
-> protocol / host / port / Path / Query

3. 스프링에서  @Controller의 의미
클라이언트 요청하면 주소를 받아들여 어디로 갈지 분석하고 맞는 길로 연결시켜주는 역활

4. 스프링에서 RequestMapping의 의미
클라이언트의 요청(URL)에 맞는 클래스나 메서드를 연결시켜주는 어노테이션이다 

5. 스프링 컨트롤러에서   return 다음에 오는 문자열의 의미
view 이름을 지정해준다.

6. Maven  Spring 의 디렉토리 구조
www / pom.xml

7. 이미지나 정적컨텐츠의 위치
www / src / main / webapp / resources

8. 스프링에서 한글을 설정하는 방법 3가지
windows > Preferences > General > workspace > text file encoding > other > UTF-8 변경
windows > Preferences > html & css > css files & html files > Encoding > UTF-8 로 변경

9. 컨텍스트패스의 의미
웹어플리케이션을 구분하기 위한 path이다.

10. 클라이언트에서 get 방식으로 request를 발생하는 방법 2가지
브라우저 주소창에 get을 직접요청, 
javascript의 ajax를 사용하여 비동기적으로 get에 요청
